Post请求和Get请求的一个区别

  • Get请求 - 由doGet接受请求

    • 客户端数据传送的方式

      表达式中的数据是直接暴漏在URl地址后面，所有的get方式的提交都是在HTTP没有消息体的

      http://localhost:8080/servlet-demos/day02/http?username=admin&pwd=123

    • 传输的数据量

      理论上数据量的长度是有限制的，但是一般不会超过2kb.因为是参数传递，并且是在地址

      栏中，所以数据是有限制的

    • 安全性 - 不安全的

  • Post请求 - 由doPost接受请求

    二进制数据的提交（图片，视频等）

    • 客户端数据传送的方式

      表单数据存放在HTTP协议的消息体中，是以实体的方式传送到服务器的

    • 传输的数据量

      适合大规模的数据传送，因为是以实体的方式进行传送的

    • 安全性 - 相对安全

    • 观察地址栏 - http://localhost:8080/servlet-demos/day03/forward

      请求的url就是一开始的url，并没有发生任何改变，只是发送了一次请求

    • 转发是一次请求

    • 转发的跳转 - 发生在服务器端 - 服务器端端跳转

  • 重定向

    • 观察地址栏 - 输入的是http://localhost:8080/servlet-demos/day03/forward

      但是地址栏的url发生了改变,变成了

      http://localhost:8080/servlet-demos/girl.jsp

    • 不会共享request作用域中的数据

    • 重定向是多次请求

    • 重定向的跳转 - 发生在客户端 - 永远是从客户端再次发送求

Cookie

  • cookie可以用来保存少量的数据，并且是通过服务器端端servlet程序把这个cookie发送到web浏览器(客户端)

    cookie是保存在客户端的

  • 客户端每次向服务器端发送请求的时候，通过请求头信息将这个cookie一起发送

    给server。

  • 一个cookie的值是用来唯一标志一个客户端的，所以cookie通常是可以用来管理session

  • cookie拥有一个name，一个单个value以及一些可选的属性(比如有效期)

  • 我们可以通过 HttpServletResponse.addCookie(javax.servlet.http.Cookie) method

    通过http的响应头信息将cookie发送到浏览器端

  • 每个客户只能为每个web服务器在本地保存20个cookie,总共可以保存300个，并且每个cookie

    的大小不能多于4KB

  • 当客户端将cookie发送到server的时候，cookie可以通过HttpServletRequest.getCookies() method

     来被检索

监听器

JavaEE包括十三种核心技术，主要的servlet和jsp技术，其中servlet规范包括三个技术点

servlet&listener&filter

什么是监听器

监听器就是监听某个对象的状态变化的组件

相关的概念

  • 事件源:被监听的对象

    在servlet中-三个作用域request,session,ServletContext

  • 监听器:监听事件源对象，事件源对象的状态的改变都会触发监听器

  • 注册监听器：将监听器和事件源进行绑定

监听器有哪些

  • 第一大类:按照被监听的对象划分:request,session,servletContext

  • 第二类:按照监听的内容，监听域对象的创建与销毁，监听域对象的属性的

    变化

ServletContext域 HttpSession域 ServletRequest域
域对象的创建与销毁 ServletContextListener HttpSessionListner ServletRequestListner
域对象内的属性的变化 ServletContextAtrributeListner HttpSessionAtrributeListner ServletRequestAtrributeListner
